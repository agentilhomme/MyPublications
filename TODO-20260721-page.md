## Design plan (from Q&A on 2026-07-21)

Answers gathered from Anais before any HTML is built:

**Overall style:** Scientific/academic — feels like a CV/journal page:
serif headings, formal, data-forward. Not minimalist, not dark-techy.

**Accent color:** Teal / cyan (works on both light and dark backgrounds).

**Layout:** Two-column — a persistent sidebar (stats + main cumulative
chart) alongside a scrollable list of papers.

**Color mode:** Both light and dark themes, with a manual toggle.

**Headline stats to feature** (in the sidebar):
- Total citations (sum across all papers)
- Paper count
- h-index
- A callout for the single most-cited paper

Computed from current data: **15 total citations**, **3 papers**,
**h-index = 2**, most-cited paper = *Mitochondrial and plastid genome
analysis of the heteromorphic red alga Mastocarpus papillatus...*
(2016, 11 citations). These will be recomputed at build time from
`data/publications.json`, not hardcoded.

**Main chart:** Cumulative citations over time (one line climbing across
years) as the primary/sidebar chart.

**Chart interactivity:** Hover shows exact year + citation count in a
tooltip; clicking a paper in the list toggles that paper's line on/off in
a per-paper comparison view (so the cumulative chart plus per-paper toggle
lines work together).

**Per-paper display:** A list (not cards/table) — each row shows the paper
info plus an inline mini-chart (sparkline) of that paper's citations
over time.

**Per-paper details shown:** venue/type badge, clickable DOI link,
co-author list, and the paper's total citation count — all four.

**Animation:** None — fast and static, no entrance or chart-draw-in
animation, prioritizing speed/simplicity.

**Responsiveness:** Desktop-first, usable on mobile, but mobile is not the
primary design target (sidebar can reasonably stack below content on
small screens without extra polish).

**Header/intro:** Name + tagline + a short bio blurb.

## Build status: index.html generated (2026-07-21)

Anais asked to proceed with the build ahead of providing final bio/tagline
text, so [scripts/build_page.py](scripts/build_page.py) generated
[index.html](index.html) with **placeholder tagline and bio text** drafted
from the papers' subject matter — these are marked `EDIT ME` at the top of
`build_page.py` (`TAGLINE`, `BIO` constants) and need review/replacement
with Anais's own words before this is published live.

Implementation notes:
- Colors follow the `dataviz` skill's validated palette: teal/cyan accent
  (`#1baf7a` light / `#199e70` dark) for the single-series cumulative
  chart and all UI chrome; the 3-paper comparison chart uses categorical
  slots 1–3 (blue/green/magenta), a prefix of the palette's pre-validated
  "first four slots pass all-pairs in both modes" set — chosen because
  `node`/the palette validator script isn't available in this environment,
  so a fresh custom ordering couldn't be re-validated.
- Headings use a serif face (Georgia) per the "scientific/academic" style
  choice; chart/data text stays in the system sans per the skill's mark
  guidance.
- Tested via a local `http.server` + direct JS execution (chart tooltip
  values, click-to-toggle, legend toggle, theme toggle, sparkline paths
  all verified correct) since the browser tool's screenshot action timed
  out in this environment; **visual appearance has not been eyeballed**,
  only functional correctness.
- Verified: cumulative chart tooltip at 2023 correctly reads 8 (matches
  hand-summed data); toggling a paper row hides its line in the
  comparison chart and marks the row inactive; theme toggle flips
  `data-theme` and persists via `localStorage`.
